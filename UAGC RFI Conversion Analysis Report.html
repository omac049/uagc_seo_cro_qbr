Executive Summary

Report Overview

This report evaluates the impact of Request For Information (RFI) optimizations implemented on June 12, 2025 across 58 key program pages on the University of Arizona Global Campus (UAGC) website. Data were pulled from Google Analytics 4 (GA4) and compare performance between the post‑change period (June 12 – July 31, 2025) and the pre‑change baseline (April 24 – June 12, 2025).

Key Objectives
  • Improve RFI form conversion rates
  • Reduce user friction in the application process
  • Enhance user experience across 58 program pages
  • Optimize "Next" button labeling and pre-selection

Faster user journey

Sessions: The selected pages received 31,821 sessions during the baseline period and 34,027 sessions post‑change — a 6.9% increase. While sessions grew modestly, the RFI changes were designed to improve efficiency rather than simply drive more traffic.

Key Events (RFIs submitted): Key events increased from 516 to 659 (27.7% increase), indicating that more visitors completed the RFI form after the optimizations.

Conversion Rate: By dividing key events by sessions, the overall conversion rate improved from 1.62% to 1.94%, a gain of 0.32 percentage points. This suggests that the RFI pre‑selection and improved “Next” button labelling successfully removed friction and encouraged visitors to submit their information.

Session Duration: Average session duration declined from 1859.5 s to 1595.9 s (–14 %). Shorter sessions accompanied higher conversion rates, meaning that visitors found what they needed faster and spent less time navigating.

Discussion

Key Insights

The RFI optimizations targeted two main friction points: auto‑selecting the program of interest when a visitor lands on a program page, and clarifying the progress indicator on the form. By eliminating the need to manually choose a program and by setting clear expectations for the number of steps involved, users were nudged to submit their information more readily.

Success Factors

The data show that these changes increased the conversion rate overall despite a slight drop in average session duration. Pages with substantial improvements tended to be those where program names are complex (e.g., health informatics analytics) or where prospective students might appreciate reassurance that they are in the right place.

Areas for Investigation

However, several pages experienced declining conversion rates. The largest declines occurred among early‑childhood and liberal‑arts programs as well as some certificate pages. This could reflect differences in user intent or the new default might have pre‑selected a program incorrectly, causing confusion.

Discussion (Legacy)

The RFI optimizations targeted two main friction points: auto‑selecting the program of interest when a visitor lands on a program page, and clarifying the progress indicator on the form. By eliminating the need to manually choose a program and by setting clear expectations for the number of steps involved, users were nudged to submit their information more readily.

The data show that these changes increased the conversion rate overall despite a slight drop in average session duration. In other words, visitors converted faster. Pages with substantial improvements tended to be those where program names are complex (e.g., health informatics analytics) or where prospective students might appreciate reassurance that they are in the right place (health human services and homeland security).

However, several pages experienced declining conversion rates. The largest declines occurred among early‑childhood and liberal‑arts programs as well as some certificate pages. This could reflect differences in user intent: for example, visitors to early‑childhood education pages might be more likely to browse multiple programs before converting. Alternatively, the new default might have pre‑selected a program incorrectly, causing confusion.

Recommendations

1
Refine RFI pre‑selection logic

Ensure that the auto‑selected program always matches the user's intent. Consider logging any instances where users change the pre‑selected program to identify mismatch patterns.

2
Custom messaging for underperforming pages

Pages such as bachelors/child‑development, associate/military‑studies and masters/education should have tailored content or micro‑copy that reassures visitors about the value of the program and encourages form submission.

3
Test alternate form designs

For the declining pages, run A/B tests with variations in the form layout, progress messaging or button labelling to determine which factors drive conversions for those audiences.

4
Monitor ongoing metrics

Continue tracking session duration, click‑through rates and drop‑off points to ensure that future enhancements do not inadvertently reduce engagement.

5
Scale successful patterns

Replicate the elements of high‑performing pages (health‑related programs) across similar pages, ensuring consistent clarity and efficient flows.

Recommendations (Legacy)

  • Custom messaging for underperforming pages: Pages such as bachelors/child‑development, associate/military‑studies and masters/education should have tailored content or micro‑copy that reassures visitors about the value of the program and encourages form submission.
  • Test alternate form designs: For the declining pages, run A/B tests with variations in the form layout, progress messaging or button labelling to determine which factors drive conversions for those audiences.
